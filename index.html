SAP S/4HANA Gateway Integration Portal

Welcome to the official integration sandbox and architectural blueprint for **Rafhan Maize Products Co. Ltd.** This portal outlines the system designs, data schemas, security architectures, and compliance models utilized to connect Rafhan's agricultural refining workflows directly with the **SAP S/4HANA Core ERP** engine.

ℹ️

API-First Architecture

Rafhan Maize avoids direct database modifications (`INSERT/UPDATE`) to protect standard S/4HANA structural rules. All data writes are executed through standard SAP NetWeaver Gateway OData services, fully preserving application audit trails.

Architecture Overview

Industrial B2B sales orders originating from export teams and third-party supplier invoices are aggregated by the middleware, validated locally for schema correctness, and securely posted via HTTPS OAuth2 directly into SAP Gateway.

SAP Digital Access Licensing Model

SAP S/4HANA enforces a modern **Digital Access** model for indirect application routing. This ensures that B2B customers placing starch or liquid glucose orders on external portals do NOT consume costly named-user licenses.

📊 Document-Based Counts

Licensing is audited strictly on the net count of core documents generated in SAP S/4HANA (e.g. Sales Orders, AP Invoices).

🔒 Secure Service Users

Connection to SAP Gateway is routed via standard role-restricted system users (`SERVICE` license types) utilizing OAuth2 scopes.

✓

Audit Compliant Setup

By registering external client order transactions under the **SAP Digital Access** framework, Rafhan Maize maintains full compliance with SAP's global licensing auditors without needing to license individual B2B client accounts.

SAP S/4HANA OData Schema Mapping

The Integration Middleware maps client payloads into standard SAP OData structure representations of the core SAP tables.

Header Mapping: Entity A_SalesOrder (Table VBAK)

Field Name OData Key SAP Type Description
Sales Order ID SalesOrder Edm.String (10) Unique document number (Primary Key)
Order Type SalesOrderType Edm.String (4) e.g., TA for Standard B2B Order
Sold-to Party SoldToParty Edm.String (10) Customer Account Code (KNA1 / BUT000)
Posting Date SalesOrderDate Edm.DateTime Posting Date of document
Doc Currency TransactionCurrency Edm.String (5) Currency code (USD, PKR, EUR)

Line Mapping: Entity A_SalesOrderItem (Table VBAP)

Field Name OData Key SAP Type Description
Line Number SalesOrderItem Edm.String (6) Item row index (e.g., 000010)
Material Code Material Edm.String (40) Refined Corn SKU (MARA table lookup)
Quantity RequestedQuantity Edm.Decimal (15,3) Requested order qty
Unit Price NetPriceAmount Edm.Decimal (11,2) Item price in Transaction Currency
Target Plant ProductionPlant Edm.String (4) Plant code (0101 Faisalabad, 0102 Jhang)

Header Mapping: Table RBKP (Incoming Invoice)

Field Name OData Key SAP Type Description
Invoice ID SupplierInvoice Edm.String (10) SAP Internal Document Number
Fiscal Year FiscalYear Edm.String (4) Posting Year (e.g., 2026)
Vendor ID InvoicingParty Edm.String (10) Supplier account code (LIFNR)
Document Date DocumentDate Edm.DateTime Invoice document date
External Ref DocumentHeaderText Edm.String (25) External vendor invoice number

Line Mapping: Table RSEG (Invoice Document Items)

Field Name OData Key SAP Type Description
Item Number SupplierInvoiceItem Edm.String (4) Line row position (e.g. 0001)
Material Code Material Edm.String (40) Raw Material SKU (e.g. Corn Grain, Chemical)
Quantity QuantityInPurchaseOrderUnit Edm.Decimal (13,3) Supplied Quantity
Net Amount SupplierInvoiceItemAmount Edm.Decimal (13,2) Net line total amount
